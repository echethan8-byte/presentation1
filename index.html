Definition

What is Dynamic Programming?

Dynamic Programming (DP) is a technique to solve problems by dividing them into overlapping subproblems, storing results, and building the final solution from those saved results. Use DP when the problem has overlapping subproblems and optimal substructure.

Key Concepts

  • Optimal substructure: an optimal solution contains optimal solutions to subproblems.
  • Overlapping subproblems: same subproblems solved multiple times.
  • Memoization (top-down): recursion + cache.
  • Tabulation (bottom-up): iterative DP table filling.

Common Problems

Fibonacci, Longest Common Subsequence (LCS), 0/1 Knapsack, Coin Change, Edit Distance, Matrix Chain Multiplication.

Code (Python)

def fib(n):
    dp = [0] * (n+1)
    dp[0] = 0
    dp[1] = 1
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

print(fib(6))  # 8

Example 2 — Longest Common Subsequence (LCS)

LCS builds a 2D table dp[m+1][n+1]. If characters match, dp[i][j]=dp[i-1][j-1]+1 else max(dp[i-1][j], dp[i][j-1]).

Quick Tips for Exams

  • Draw the DP table for small inputs — it makes reasoning easy.
  • State recurrence clearly before coding.
  • Indicate base cases (dp[0]...) on your flowchart.
  • Mention time (usually O(n * W) or similar) and space — and offer space optimization when possible.
